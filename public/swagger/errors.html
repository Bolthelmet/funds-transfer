Common HTTP status codes

Code Text Description
400 Bad Request The request was invalid or cannot be otherwise served. Make sure you sent all required parameters and the body of the request is valid. An accompanying error message will explain further.
401 Unauthorized Authentication credentials were missing or incorrect. Make sure you have API key and session ID sent in the request.
403 Not Found The request is understood, but it has been refused or access is not allowed. An accompanying error message can explain further.
500 Internal Server Error Something is broken in the server side and client cannot do anything to fix the error.

Error Messages

Error messages are provided in JSON format. For example, an error might look like this:

{"code": 1, "msg": "Client update required"}

Error Codes

In addition to descriptive error text, error messages contain machine-parseable codes. While the text for an error message may change, the codes will stay the same. If you see an response which is not listed in the above table, then fall back to the HTTP status code in order to determine the best way to address the error.

Code Text Description
0 Common error Usually no more details are available because client cannot do anything to fix the error.
1 Client update required Version of the client is not supported and it must be updated.
2 Invalid session ID Session ID is invalid or it is expired. Client must provide a valid session ID.
3 Invalid API key API key is invalid. Client must provide a valid API key.
4 Invalid JSON Provided JSON payload is invalid. Client must check the request and send a valid payload.
